## 1. Цель проекта

Разработать прототип системы мониторинга из 5 видов датчиков на базе микроконтроллера **ESP32** с передачей данных на локальный сервер по защищённым каналам связи.

## 2. Задачи проекта

- Проектирование аппаратной части (обвязка ESP32 с датчиками)
- Разработка прошивки для микроконтроллера
- Разработка протокола обмена данными
- Обеспечение информационной безопасности (шифрование трафика)
- Тестирование и проверка работоспособности системы

## 3. Требования к аппаратной части

| №   | Датчик                                      | Тип        |
| --- | ------------------------------------------- | ---------- |
| 1   | DS18B20 - датчик температуры                | Аналоговый |
| 2   | фоторезистор                                | Аналоговый |
| 3   | ультразвуковой датчик расстояния<br>HC-SR04 | Цифровой   |
| 4   | RC522 — считыватель NFC/RFID                | Цифровой   |
| 5   | Датчик открытия/закрытия                    | Дискретный |

## 4. Функциональные требования

### 4.1. Сбор и передача данных

- Автоматическая периодическая передача данных на сервер
- Интервал настраивается (N секунд/минут)
- Протокол передачи: MQTTS
- ==Среда передачи==

### 4.2. Безопасность

- Шифрование трафика: SSL/TLS
- Подключение по протоколу MQTTS
- Защита от перехвата и несанкционированного доступа

### 4.3. Сигнализация

- Мгновенная отправка тревожного сообщения при выходе параметров за пороговые значения
- Срабатывание вне зависимости от основного цикла телеметрии
- Возможность удалённой или локальной настройки порогов

### 4.4. Интерактивность (запрос-ответ)

- Сервер может отправить команду конкретному датчику
- ESP32 обрабатывает входящие команды и отвечает актуальными показаниями

### 4.5. Самодиагностика

- Отслеживание зависаний программы (watchdog)
- Периодическая отправка подтверждения работоспособности (heartbeat)
- Проверка физического подключения датчика к микроконтроллеру

## 5. Требования к электропитанию

- USB (для разработки и отладки)
- Внешний источник питания: **5V / 3.3V**

## 6. Этапы реализации

1. **Анализ и подбор компонентов** — уточнение моделей датчиков, выбор схемы подключения
2. **Проектирование схемы** — разработка электрических схем подключения датчиков к ESP32
3. **Разработка прошивки**
    - Написание кода на языке C
    - Реализация MQTTS-клиента с шифрованием TLS
    - Реализация механизма обработки команд
4. **Настройка серверной инфраструктуры** — установка и конфигурация локального сервера
5. **Интеграционное тестирование** — проверка связки «Датчик — Сервер — Оповещение»
6. **Документирование** — составление инструкции по подключению и настройке

## 7. Требования к документации

- Электрические схемы подключения
- Исходный код прошивки с комментариями
- Описание реализации шифрования и настройки локальной сети

## 8. Критерии приёмки

- [ ] Все 5 датчиков успешно регистрируются на сервере и передают данные
- [ ] Трафик между датчиком и сервером шифруется посредством TLS
- [ ] При превышении порогового значения сервер получает аварийное сообщение не позднее чем через **2 секунды**
- [ ] Сервер может отправить команду датчику, датчик незамедлительно отвечает актуальными показаниями
- [ ] При потере связи с датчиком система диагностики фиксирует факт отключения
